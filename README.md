# common library chart pattern

A "write-once, use-many" Helm setup: all Kubernetes resource logic lives in
the `common` library chart (published as an OCI artifact to GHCR). Every
microservice chart becomes just three files:
`Chart.yaml`, `values.yaml`, and a 9-line `templates/all.yaml`.

## Structure

```
.
├── common/                          # library chart (type: library)
│   ├── Chart.yaml
│   └── templates/
│       ├── _helpers.tpl             # naming, labels, image, secret/pvc names
│       ├── _configmap.tpl           # common.configmap
│       ├── _deployment.tpl          # common.deployment
│       ├── _externalsecret.tpl      # common.externalsecret (ESO)
│       ├── _hpa.tpl                 # common.hpa
│       ├── _ingress.tpl             # common.ingress
│       ├── _pvc.tpl                 # common.pvc
│       ├── _service.tpl             # common.service
│       ├── _serviceaccount.tpl      # common.serviceaccount
│       └── _virtualservice.tpl      # common.virtualservice (Istio)
│
├── .github/workflows/
│   └── publish-common.yml           # helm package + push -> oci://ghcr.io/.../helm-chart
│
│                     # example microservice chart
├── Chart.yaml                   # dependencies -> common (OCI)
├── Chart.lock                   # auto-generated by helm dependency update
├── values.yaml                  # ALL values consumed by common.* templates
└── templates/
   └── all.yaml                 # 9 lines, one per common.* include
```

## One-time setup

1. Push `common/` to GHCR:
   ```bash
   git push origin main   # triggers .github/workflows/publish-common.yml
   ```
   Then set the resulting package (`helm-chart/common`) to **Public** under
   your GitHub org's Packages settings, or configure pull credentials for
   private access.

## Per-microservice workflow

```bash
# 1. Resolve & lock the common dependency
helm dependency update ./pg-fds--be
#    -> writes pg-fds--be/Chart.lock and downloads
#       pg-fds--be/charts/common-1.0.1.tgz

# 2. Lint & render
helm lint ./pg-fds--be
helm template pg-fds-be ./pg-fds--be -f values-prod.yaml

# 3. Install / upgrade
helm upgrade --install pg-fds-be ./pg-fds--be \
  --namespace production --create-namespace \
  -f values-prod.yaml
```

## Bumping common for everyone

1. Edit a `common/templates/_*.tpl` file (e.g. add a new field to
   `common.deployment`).
2. Bump `version` in `common/Chart.yaml`.
3. Push to `main` -> new OCI tag published.
4. In each microservice chart, bump the `common` version in `Chart.yaml`
   and run `helm dependency update`.

## Feature matrix (all driven by values.yaml, zero template edits)

| Feature                      | values.yaml key                     | Backing resource               |
| ---------------------------- | ----------------------------------- | ------------------------------ |
| App config                   | `configMap.*`                       | ConfigMap                      |
| Secrets                      | `externalSecret.*`                  | ExternalSecret (ESO) -> Secret |
| Persistent storage           | `persistence.*`                     | PersistentVolumeClaim          |
| HTTP ingress (nginx/alb/etc) | `ingress.*`                         | Ingress                        |
| Service mesh routing         | `virtualService.*`                  | Istio VirtualService           |
| Autoscaling                  | `autoscaling.*`                     | HorizontalPodAutoscaler        |
| Extra containers             | `initContainers`, `sidecars`        | Deployment spec                |
| Extra volumes                | `extraVolumes`, `extraVolumeMounts` | Deployment spec                |
